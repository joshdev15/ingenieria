#engineering #project-management

El modelo híbrido combina enfoques ágiles y tradicionales para adaptar la metodología al proyecto.

## Cuándo Usar Híbrido

- Proyectos con partes definidas y partes cambiantes
- Equipos con experiencia mixta
- Necesidad de estructura + flexibilidad
- Requisitos regulatorios que necesitan documentación

## Ejemplos de Combinación

### Scrum + Waterfall
```
Fase waterfall: Requisitos, Diseño
Fase Scrum: Desarrollo iterativo
Fase waterfall: Pruebas, Despliegue
```

### Scrum + PRINCE2
- PRINCE2 para estructura de proyecto
- Scrum para gestión de iteraciones
- Roles de PRINCE2 (Executive) + Scrum (PO)

### Kanban + Waterfall
- Kanban para equipos de soporte
- Waterfall para proyectos de infraestructura

## Ventajas

- Flexibilidad para adaptarse
- Combina fortalezas de ambos enfoques
- Puede funcionar en diferentes partes del proyecto

## Desafíos

- Puede crear confusión de roles
- Necesita definir qué es ágil vs tradicional
- Requiere experiencia del equipo

## Mejores Prácticas

1. Definir claramente qué será ágil vs tradicional
2. Mantener comunicación constante
3. Ajustar roles según necesidad
4. Medir resultados y adaptar
