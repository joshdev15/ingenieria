#engineering #project-management

Waterfall (Modelo en Cascada) es una metodología secuencial lineal donde cada fase debe completarse antes de iniciar la siguiente.

## Características

- Enfoque secuencial
- Progresivo (no hay retrocesos)
- Documentación exhaustiva
- Fases bien definidas

## Fases

| Fase | Descripción |
|------|-------------|
| **1. Requisitos** | Recopilar y documentar todos los requisitos |
| **2. Diseño** | Crear arquitectura y diseño del sistema |
| **3. Implementación** | Desarrollar el código |
| **4. Verificación** | Pruebas del sistema |
| **5. Mantenimiento** | Despliegue y soporte |

## Flujo

```
Requisitos → Diseño → Implementación → Verificación → Mantenimiento
    ↓           ↓            ↓             ↓              ↓
  (Siguiente fase comienza al completar la anterior)
```

## Ventajas

- Simple de entender y gestionar
- Fases bien definidas
- Documentación completa
- Adecuado para requisitos estables

## Desventajas

- Difícil incorporar cambios
- No hay entregas tempranas
- Riesgo alto si hay errores tardíos
- Cliente ve el producto al final

## Cuándo Usar

- Requisitos muy claros y estables
- Proyecto pequeño/mediano
- Tecnologia bien conocida
- Contratos de precio fijo

## Comparación con Ágil

| Aspecto | Waterfall | Ágil |
|---------|-----------|------|
| Flexibilidad | Baja | Alta |
| Entregas | Al final | Incremental |
| Requisitos | Fijos | Cambian |
| Riesgos | Tardíos | Tempranos |
