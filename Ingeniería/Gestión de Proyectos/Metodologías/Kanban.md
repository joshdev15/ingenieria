#engineering #project-management #agile

Kanban es un método de gestión de proyectos visuales basado en el flujo de trabajo.

## Principios Fundamentales

### 1. Visualizar el Trabajo
Usar un tableau (tablero) para ver todas las tareas.

### 2. Limitar el WIP (Work in Progress)
Definir límites de tareas en progreso por columna.

### 3. Gestionar el Flujo
Medir y optimizar el tiempo de entrega.

### 4. Hacer Políticas Explícitas
Definir reglas claras del proceso.

### 5. Implementar Ciclos de Retroalimentación
Reuniones regulares para mejorar.

### 6. Mejorar Colaborativamente
Evolución incremental del proceso.

## Tablero Kanban

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│   To Do     │   In Progr. │   Review    │    Done     │
├─────────────┼─────────────┼─────────────┼─────────────┤
│  Task 1     │   Task 3    │   Task 5    │   Task 2    │
│  Task 4     │             │             │   Task 6    │
│             │   Limit: 2  │   Limit: 2  │             │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

## Métricas

| Métrica | Descripción |
|---------|-------------|
| **Lead Time** | Tiempo desde solicitud hasta entrega |
| **Cycle Time** | Tiempo en ejecución |
| **Throughput** | Tareas completadas por período |
| **WIP** | Tareas en progreso |

## Diferencias con Scrum

| Aspecto | Scrum | Kanban |
|---------|-------|--------|
| Iteraciones | Fijas (sprints) | Continuo |
| Roles | Definidos | Opcionales |
| Cambios | Al final del sprint | Cuando sea necesario |
| Planificación | Por sprint | Continua |

## Implementación

1. Crear un tablero con columnas de tu flujo
2. Añadir todas las tareas como tarjetas
3. Establecer límites WIP
4. Medir lead time y cycle time
5. Mejorar iterativamente
