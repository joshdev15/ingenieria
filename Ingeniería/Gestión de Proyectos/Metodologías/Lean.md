Lean (Lean Software Development) aplica principios de manufactura Toyota al desarrollo de software.

## Principios Fundamentales

1. **Eliminar Desperdicio**
   - Código no usado
   - Procesos innecesarios
   - Esperas entre tareas
   - Funcionalidad no requerida

2. **Maximizar el Valor**
   - Focus en features que aportan valor
   - Priorizar según impacto de negocio

3. **Aprendizaje Rápido**
   - Ciclos cortos de feedback
   - Experimentar y adaptar

4. **Decisiones Retardadas**
   - Mantener opciones abiertas
   - Decide lo más tarde posible

5. **Entrega Rápida**
   - Mínimo producto viable
   - Iteraciones pequeñas

6. **Respeto al Equipo**
   - Auto-organización
   - Empoderar a las personas

## Herramientas

| Herramienta | Uso |
|------------|-----|
| **Value Stream Mapping** | Visualizar flujo de valor |
| **5S** | Orden y limpieza |
| **Kaizen** | Mejora continua |
| **Kanban** | Flujo de trabajo |

## 7 Tipos de Desperdicio (MUDA)

1. Trabajo parcialmente hecho
2. Características extras
3. Aprendizaje no utilizado
4. Transferencias (handoffs)
5. Esperas
6. Tareas duplicadas
7. Defectos

## Relación con Agile

Lean complementa metodologías ágiles con:
- Mayor énfasis en eliminación de desperdicio
- Focus en flujo de valor
- Mejora continua sistemática
