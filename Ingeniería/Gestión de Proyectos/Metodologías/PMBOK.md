#engineering #project-management

PMBOK (Project Management Body of Knowledge) es una guía de estándares para la gestión de proyectos desarrollada por el PMI.

## Grupos de Procesos

| Grupo | Descripción |
|-------|-------------|
| **Iniciación** | Definir proyecto, obtener autorización |
| **Planificación** | Definir alcance, planificar esfuerzo |
| **Ejecución** | Coordinar personas y recursos |
| **Monitoreo y Control** | Seguimiento y control de progreso |
| **Cierre** | Finalizar todas las actividades |

## Áreas de Conocimiento

| Área | Descripción |
|------|-------------|
| **Integración** | Coordinar todos los procesos |
| **Alcance** | Definir qué se incluye/excluye |
| **Cronograma** | Gestionar tiempos |
| **Costo** | Gestionar presupuesto |
| **Calidad** | Asegurar estándares |
| **Recursos** | Gestionar equipo y materiales |
| **Comunicaciones** | Información oportuna |
| **Riesgos** | Identificar y responder |
| **Adquisiciones** | Compras y contratos |
| **Stakeholders** | Involucrar partes interesadas |

## Entradas, Herramientas, Salidas (EHS)

Cada proceso sigue la estructura:
- **Entradas**: Inputs del proceso
- **Herramientas**: Técnicas utilizadas
- **Salidas**: Resultados del proceso

## Certificaciones

| Certificación | Descripción |
|----------------|-------------|
| **CAPM** | Certified Associate in Project Management |
| **PMP** | Project Management Professional |
| **PgMP** | Program Management Professional |

## Relación con otras metodologías

PMBOK es una guía de mejores prácticas neutral, puede combinarse con:
- Scrum (para desarrollo ágil)
- PRINCE2 (para estructura)
- Waterfall (para proyectos tradicionales)
