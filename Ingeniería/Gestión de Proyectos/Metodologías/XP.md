#engineering #project-management #agile

XP (Extreme Programming) es una metodología ágil centrada en la calidad del código y la satisfacción del cliente.

## Valores

- **Comunicación**: Equipo cercano y comunicativo
- **Simplicidad**: Solo hacer lo necesario
- **Retroalimentación**: Feedback rápido y continuo
- **Coraje**: Facing cambios y desafíos
- **Respeto**: Valorar al equipo y al cliente

## Prácticas Principales

| Práctica | Descripción |
|----------|-------------|
| **Planning Game** | Planificar historias de usuario |
| **Customer Team** | Cliente integrado en el equipo |
| **Pair Programming** | Dos programadores en una máquina |
| **Test-Driven Development (TDD)** | Primero tests, luego código |
| **Continuous Integration** | Integrar código frecuentemente |
| **Refactoring** | Mejorar código continuamente |
| **Simple Design** | Diseño lo más simple posible |
| **Collective Ownership** | Cualquiera puede modificar código |

## Ciclos (Iteraciones)

Cada iteración (1-2 semanas) incluye:
1. Seleccionar historias del backlog
2. Estimar effort
3. Desarrollar en pares
4. Tests primero
5. Integrar continuamente
6. Demo al cliente

## Diferencias con Scrum

| Aspecto | XP | Scrum |
|---------|-----|-------|
| Focus | Calidad de código | Entregas de valor |
| Estimación | Story points + effort | Story points |
| Roles | + engineer practices | Roles definidos |
| Testing | TDD obligatorio | Opcional |
